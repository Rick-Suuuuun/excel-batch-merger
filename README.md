# Excel 批量合并、去重与来源标记工具

## 项目简介

这是一款面向日常办公场景的 Excel 数据整理工具。它可以一次读取多个 `.xlsx` 文件，自动识别主流表头、排除不符合要求的文件，并将有效数据合并到一张表中。

合并后的每条记录都会保留来源文件名，方便后续核对和追溯。用户还可以根据订单号、手机号等业务字段进行去重，减少手工复制、筛选和检查带来的时间成本与操作风险。

工具同时提供简单图形界面和命令行版本，不会修改任何原始 Excel 文件。

## 适用场景

- 合并多个门店、部门或业务人员提交的同格式报表
- 汇总订单、客户、报名、销售或库存数据
- 清理多个文件中的重复订单、重复手机号或重复客户记录
- 在汇总数据中保留来源，便于追踪问题文件
- 快速检查一批 Excel 文件是否使用了统一表头

## 核心功能

- 批量扫描指定输入目录中的 `.xlsx` 文件
- 自动跳过 Excel 产生的 `~$` 临时文件
- 默认读取每个文件的第一个工作表
- 自动识别空表并记录跳过原因
- 统计不同表头的出现次数，以出现次数最多的表头作为标准
- 多种表头并列时，由用户手动选择标准表头
- 只合并表头与标准表头完全一致的文件
- 为每行数据增加“来源文件”列
- 支持单字段或多字段去重，并保留第一次出现的记录
- 自动创建输出目录并生成对应的结果文件
- 在终端展示成功、跳过、失败、行数及去重统计
- 提供简单图形界面，可选择输入文件夹并查看处理结果

## 安装方法

建议使用 Python 3.10 或更高版本。

1. 打开终端并进入项目目录。
2. 可选：创建独立的 Python 虚拟环境。

```powershell
python -m venv .venv
.venv\Scripts\activate
```

macOS 或 Linux 用户可使用：

```bash
source .venv/bin/activate
```

3. 安装项目依赖。

```powershell
pip install -r requirements.txt
```

## 使用方法

### 图形界面（推荐）

在项目根目录运行：

```powershell
python gui.py
```

点击“选择文件夹”，选择包含 Excel 文件的目录；按需填写去重字段，然后点击“开始合并”。完成后，界面会显示处理数量、行数和输出文件路径。

### 命令行版本

1. 将需要处理的 Excel 文件放入：

```text
sample_data/input/
```

2. 在项目根目录运行：

```powershell
python main.py
```

3. 程序会自动统计表头并展示可合并文件与跳过文件。如果最高频表头出现并列，按照终端提示输入序号即可。

4. 合并完成后，程序会询问是否去重：

```text
是否需要去重？请输入去重字段（多个字段用逗号分隔，直接回车不去重）：
```

5. 输入去重字段，或者直接按回车保留全部数据。处理完成后，结果会保存到 `output` 文件夹。

## 输入文件要求

- 文件格式必须为 `.xlsx`，暂不支持旧版 `.xls`
- 图形界面可选择任意输入文件夹；命令行版本默认读取 `sample_data/input`
- 当前不会扫描所选目录下的子文件夹
- 每个文件只读取第一个工作表
- 第一行应为表头，后续行为数据
- 需要参与合并的文件，其表头名称和列顺序必须完全一致
- 表头前后的多余空格会自动清理
- 空表会被跳过，并在终端显示原因
- `~$` 开头的 Excel 临时文件会被自动忽略
- 文件不能处于加密或损坏状态

## 去重字段说明

去重不是必选操作。直接回车时，程序会导出全部合并数据。

支持输入一个字段：

```text
订单号
```

也支持输入多个字段，英文逗号和中文逗号均可：

```text
订单号,手机号
```

多字段去重时，只有这些字段的组合值全部相同，才会被判断为重复记录。程序保留第一次出现的数据，删除后续重复数据，“来源文件”列仍会保留。

字段名称必须与表头一致。如果输入不存在的字段，程序会显示可用字段列表，并提示重新输入。

## 输出文件说明

所有结果均保存到：

```text
output/
```

输出文件名会根据去重方式自动生成：

| 处理方式 | 输出文件 |
| --- | --- |
| 不去重 | `合并结果_未去重.xlsx` |
| 按手机号去重 | `合并结果_按手机号去重.xlsx` |
| 按订单号去重 | `合并结果_按订单号去重.xlsx` |
| 按订单号和手机号去重 | `合并结果_按订单号_手机号去重.xlsx` |

如果同名文件已经存在，程序会直接覆盖。输出表第一行为标准表头，最后一列为“来源文件”。

## 常见问题

### 为什么有些文件没有参与合并？

文件可能是空表、表头与标准表头不一致，或者无法正常读取。程序会在终端列出文件名和具体原因。

### 为什么程序选择了这个标准表头？

程序会统计所有非空文件的表头，优先选择出现次数最多的一种。如果多种表头出现次数相同，会请用户手动选择。

### 为什么输入去重字段后提示字段不存在？

请确认输入内容与表头名称一致。程序会同时显示当前可用字段，可直接参照重新输入。

### 为什么去重后记录数量比预期少？

程序会将指定字段值相同的记录视为重复，并保留第一次出现的记录。使用多个字段可以让判断条件更精确。对于空白去重字段，也建议先检查原始数据是否符合业务预期。

### 为什么无法生成或覆盖输出文件？

请先确认目标 Excel 文件没有在其他程序中打开。文件被 Excel 占用时，系统可能不允许覆盖。

### 工具会修改原始文件吗？

不会。程序只读取输入文件，并将处理结果写入独立的 `output` 文件夹。

## 暂不支持的功能

- 扫描多层子文件夹
- `.xls`、`.csv` 等其他文件格式
- 同时合并一个文件中的多个工作表
- 保留复杂格式、图表、图片、公式样式和宏
- 处理加密 Excel 文件
- 针对百万行超大文件的专门优化
- 打包为独立的 Windows 可执行程序
